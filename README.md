# File Search Explorer

# 【重要】不具合に関するお詫びと修正版について
以前公開しておりましたv0.2およびv0.2betaにおいて、ダークモード使用時に検索結果の一部が
白く表示されてしまう不具合が確認されました。ご利用中の皆様には多大なご迷惑をおかけし、
深くお詫び申し上げます。

v0.2修正版にて、該当の不具合を修正しました。あわせて、ダークモードの
配色の塗り漏れ(スクロールバー・ドロップダウンリスト・右クリックメニューなど)や、
検索速度の改善も行っています。安心してご利用ください。

タブで探す、圧倒的な速さで見つかる。
ローカル完結のファイル検索デスクトップアプリです。

Windows標準の検索よりも柔軟で、インデックス機能により2回目以降の検索が瞬時に終わります。

## ダウンロード

最新版は [Releases](../../releases) から入手できます。

ダウンロードするファイルは「FileSearchExplorer_(バージョン名).zip」です。
「Source code」と書かれた2つのリンクはGitHubが自動生成するもので、
アプリの実行には使用しません。

配布物には実行ファイル(.exe)のみを含めており、ソースコード一式はReleaseの
添付ファイルとしては公開していません(リポジトリ本体の閲覧は可能です)。

## 特徴

### タブで並行検索
複数のフォルダを同時に検索したいとき、タブを増やすだけ。Explorer のタブ感覚のまま、切り替えながら検索できます。

### 複数フォルダ同時検索 (v0.6.5〜)
1つのタブ内でも、複数のフォルダをまとめて検索できるようになりました。Everything風に、
異なるドライブ・異なる場所のフォルダを1回の検索でまとめて確認できます。
- 「検索フォルダ」欄の下にある「複数フォルダ検索」欄からフォルダを追加すると、そちらが
  優先され、追加した全フォルダをまとめて検索します
- 追加したフォルダは一覧からいつでも「− 削除」で個別に外せます。「全消去」で一覧を
  空にすると、従来通り上の「検索フォルダ」欄1件だけの検索に戻ります
- タブを使った並行検索と組み合わせて使うことも可能です

### インデックス機能
- 一度インデックスを作成しておけば、2回目以降の検索は瞬時
- 検索したフォルダは自動でバックグラウンドインデックス化
- 登録済みフォルダは起動時・30分ごとに自動で最新状態に更新
- インデックスの更新(自動・手動とも)は、変更・新規のファイルだけをデータベースへ書き込み、
  検索から消えたファイルだけを削除する差分更新方式に変更。以前は更新のたびに対象フォルダの
  登録内容をすべて削除してから作り直していましたが、SSDへの書き込み負荷が減り、更新時間も
  大幅に短縮されました (v0.6.5〜)
- 作業の邪魔をしない、静かなバックグラウンド処理
- キーワード検索はSQL側でも絞り込みを行い、件数が多いインデックスでも高速に動作
  (日本語のあいまい検索を使う場合を除く。詳しくは後述の「あいまい検索・日本語検索の
  強化」をご覧ください)
- インデックス管理画面からデータベースの最適化(VACUUM)も可能
- インデックス管理画面に、登録フォルダ全体の合計件数・データ量をまとめて表示 (v0.5〜)
- 実際には存在しないフォルダをまとめてインデックスから削除できるボタンを追加 (v0.5〜)
- サイズ・更新日時での絞り込みもインデックス側で対応 (v0.6〜。作成日時での絞り込みのみ、
  インデックス未対応のため自動的に通常検索にフォールバックします)
- 【不具合修正】検索フォルダにドライブ直下(C:\ など)を指定してインデックス検索すると、
  サブフォルダ内のファイルが一切ヒットせず、直下のファイル・フォルダしか表示されない
  不具合を修正しました。通常のフォルダを指定した場合は影響ありません (v0.6.7〜)

### 柔軟な検索オプション
- 部分一致・完全一致検索
- 拡張子フィルタ(画像・動画・文書・音楽・圧縮ファイルなどのカテゴリからワンクリックで追加可能)
- 除外拡張子フィルタ(特定の拡張子を検索結果から除外)
- 隠し/システムファイルの表示切り替え
- フォルダ名も検索対象にできる

### あいまい検索・日本語検索の強化 (v0.6.6〜)
「だいたい合っていれば見つかる」検索を強化しました。
- あいまい検索(誤字補正)の精度を向上。「ython」と入力しても「python」がヒットするなど、
  1文字程度の入力ミス(誤字・脱字)であれば、編集距離による判定を追加したことで確実に
  ヒットするようになりました
- 日本語のあいまい検索に対応。「東京都」で検索すると「東京」や「都」を含むファイルも
  ヒットするほか、「とうきょう」「トウキョウ」のようにひらがな・カタカナで検索しても、
  漢字表記の「東京」がヒットします(通常の検索やあいまい検索で見つからなかった場合の
  補助として働きます)
- 上記の日本語あいまい検索は、設定画面からON/OFFを切り替え可能です。形態素解析ライブラリ
  「janome」を利用しており、未導入環境では自動的に無効になります

### 検索の挙動をカスタマイズ (v0.6.6〜)
設定画面から、検索まわりの挙動を好みに合わせて切り替えられるようになりました。
- リアルタイム検索: インデックス検索が有効なとき、キーワードなどを入力するたびに
  自動で再検索するかどうかを切り替え可能(既定はOFF)
- キーワード欄の候補一覧: 欄の▾ボタンを押したときに、検索履歴(新しいものから順)に
  加えてインデックス内のファイル名・フォルダ名も候補として表示するかどうかを切り替え可能
  (既定はON)

### 詳細検索 (v0.6〜)
検索条件エリアの「詳細検索」パネルから、より細かい条件で絞り込めます。
(v0.6.4〜、拡張子/除外拡張子の指定や、お気に入り・最近のフォルダ・保存した検索も
このパネルに統合されました)
- キーワードの一致方法を 部分一致 / 完全一致 / 正規表現 / あいまい検索(誤字・表記ゆれを
  許容する類似度判定) から選択可能
- キーワード欄に AND / OR / NOT を書くことで複数語句を組み合わせて検索可能
  (例: `見積 AND 2024 NOT 下書き`。正規表現モードでは無効で、パターン全体を1つの
  正規表現として扱います)
- 除外フォルダ指定(カンマ区切りのフォルダ名。例: `node_modules,.git`)
- ファイルサイズ範囲検索(単位はB/KB/MB/GBから選択)
- 更新日時範囲検索・作成日時範囲検索(YYYY-MM-DD形式)
  - 作成日時での絞り込みはインデックス検索が非対応のため、指定時は自動的に通常検索へ
    切り替わります
  - macOS/Linuxでは、OSの仕様上「作成日時」が正確に取得できない場合があります
- ファイル種類ごとの検索テンプレート(画像・動画・音楽・文書・圧縮ファイル・実行ファイル・
  コード)を選ぶと、拡張子・内容検索の有無・フォルダ検索対象の設定を一括で適用

### シンプルな操作画面 (v0.6.4〜)
メイン画面には「検索フォルダ」「キーワード」「よく使うチェック2つ(サブフォルダも検索/
インデックスを使用する)」「検索ボタン」だけを表示するようにしました。拡張子/除外拡張子の
指定欄、お気に入り・最近のフォルダ・保存した検索条件、そのほかのチェックボックスは、
「詳細検索」パネルにまとめてあります。
- 検索ボタンを大きく目立つデザインに変更
- 検索中の進捗をプログレスバーで表示。見つかった件数・経過時間もリアルタイムに表示
  (v0.6.7〜、件数・経過時間の表示位置をプログレスバーの右からバーの下の行に変更)
- 結果クリア・エクスポート・権限エラー表示のボタンは、結果一覧のすぐ上(絞り込み欄の隣)に移動
- 各項目の余白を詰め、結果一覧がより広く表示されるようレイアウトを調整 (v0.6.6〜)
- 検索フォルダ・キーワード・チェックボックス・各種ボタンなど、ほぼすべての操作項目に
  マウスを乗せると説明が表示されるツールチップを追加。初めてお使いの方でも迷ったときに
  すぐ確認できます (v0.6.6〜)
- ブランドカラー(青)をボタン・タブ・結果一覧の見出しなどに取り入れ、見た目に統一感を
  持たせました (v0.6.6〜)

### 重複ファイル検索 (v0.6〜)
指定フォルダ以下から、内容が完全に同一のファイル(重複ファイル)を検出します。
- まずファイルサイズでグループ分けし、サイズが一致するファイルだけSHA-256ハッシュを
  計算する2段階方式で、無駄な計算を抑えて高速に判定
- 最小サイズ・拡張子での絞り込みに対応
- 重複グループごとに一覧表示。削除すれば節約できる容量の目安も確認可能
- 選択したファイルを開く／フォルダを開く／パスをコピー／完全に削除、が可能
  (削除は元に戻せません。削除前に必ず内容をご確認ください)
- 削除実行時は、赤系の警告色の確認ダイアログと警告音で誤操作を防止。システム関連の
  フォルダ(Windows、Program Files、Program Files (x86)、ProgramData)は、誤って
  重要なファイルを削除してしまうことがないよう、検索対象自体に指定できません (v0.6.2〜)
- ツールバーの「🧬 重複ファイル検索」、またはメニューから起動できます

### ファイル内容検索 (v0.4〜)
ファイル名にキーワードが含まれていなくても、テキスト系ファイル(txt・py・md・
csv・jsonなど)の中身にキーワードが含まれていれば検索結果に表示できます。
- チェックボックスでON/OFFを切り替え可能(OFFなら従来通りファイル名検索のみ)
- ヒットした行番号と該当箇所の抜粋を結果一覧に表示
- 巨大ファイル(5MB超)やテキスト系以外の拡張子は自動的に対象外
- インデックス検索でも内容検索に対応(v0.6.1〜。まずインデックスでファイル名により
  絞り込み、ファイル名で一致しなかったものだけ内容を照合するハイブリッド方式のため、
  高速に動作します)

### アクセスできなかったフォルダの一覧表示 (v0.6.1〜、v0.6.3で高速化)
検索対象フォルダ以下に、権限がない・移動/削除されたなどの理由でアクセスできない
フォルダがあった場合、種別・パス・詳細を一覧表示します。ツールバーの「権限エラー」
ボタンからいつでも確認・クリアできます。
- v0.6.3〜、検索を始める前に専用のチェックを行う方式を廃止しました。通常検索は
  実行した検索そのものの走査結果から、インデックス検索は最後のインデックス作成時点の
  情報から一覧を表示するため、検索開始が待たされることはありません
- 一覧ダイアログには情報がいつ時点のものか(今回の検索結果／インデックス作成時点など)
  もあわせて表示。「今すぐ再確認」ボタンを押せば、その場で最新のアクセス状況を
  強制的に確認することもできます

### 検索完了通知 (v0.6.1〜)
検索が完了した際に、通知音・デスクトップ通知(OSのトースト通知)・アプリ内
ポップアップでお知らせできます。設定画面からそれぞれON/OFFを切り替え可能です。
- 通知音はWindows内蔵のサウンド、またはお手持ちのwavファイルのどちらかを選択可能
  (v0.6.2〜、対応形式をwavファイルのみに変更しました)
- デスクトップ通知が利用できない環境では、該当項目が自動的に無効化されます
- デスクトップ通知(Windows通知)の既定値をOFFに変更しました。検索完了は通知音と
  アプリ内ポップアップでお知らせします。Windows通知も併用したい場合は、設定画面から
  いつでもONに切り替えられます (v0.6.5〜)

### 検索速度履歴 (v0.6.2〜)
過去の検索所要時間・件数を履歴として自動保存し、専用の「検索速度履歴」画面で
一覧・確認できます。検索完了時には、同じフォルダへの前回検索との速度比較
(例: 「前回(2.0秒)より50%高速化」)もあわせてステータスバー・通知に表示されます。
インデックス作成の効果を体感しやすくなりました。

### 結果一覧の絞り込み・並び替え・表示列カスタマイズ (v0.5〜)
- 検索をやり直さず、その場で結果一覧を絞り込める「結果を絞り込み」欄を追加。
  ファイル名・フォルダ・内容一致箇所を対象に、入力するたびにリアルタイムで
  表示件数が絞り込まれます(表示件数/全件数も表示)
- 名前・フォルダ・サイズ・更新日時の列見出しをクリックすると昇順・降順で並び替え
- 並び替えた列・順序(昇順/降順)はアプリを閉じても保存され、次回起動時にも
  自動的に復元されます (v0.6.5〜)
- 結果一覧の「☰ 表示列」ボタンから、フォルダ・サイズ・更新日時・内容一致箇所の列を
  個別に表示/非表示切り替え可能。列幅はアプリ終了時に自動保存され、次回起動時にも
  復元されます (v0.6.2〜)

### 検索条件の保存
フォルダ・キーワード・拡張子・除外拡張子・各種チェックボックスなど、検索条件を
まとめて名前を付けて保存し、プルダウンからワンクリックで呼び出せます。よく使う
検索条件がある場合にご活用ください。(v0.5〜。v0.6〜、上記「詳細検索」の条件も
あわせて保存・呼び出しできるよう拡張しました)
- 保存した検索条件は、ファイルメニューからJSON形式でエクスポート/インポートも
  可能です。バックアップや、他のPCへの検索条件の引き継ぎにご活用いただけます (v0.6.2〜)

### ドラッグ&ドロップ (v0.5〜)
エクスプローラーからフォルダ(またはファイル)をアプリの画面にドラッグ&ドロップ
すると、検索フォルダに設定して自動的に検索を開始します
(環境によっては本機能が無効になる場合がありますが、その場合も他の機能には影響ありません)。

### 開く前の確認・右クリックメニュー (v0.4〜)
検索結果のファイルをダブルクリックまたは「開く」で開こうとした際は、誤操作防止のため
必ず確認ダイアログを表示するようにしました。exe・bat・ps1・vbsなどの実行ファイル/
スクリプトの場合は、より強い警告文言つきのダイアログを表示します
(フォルダを開く操作やプレビュー表示には影響しません)。
- 右クリックメニューから、パスに加えてファイル名・更新日時・サイズも個別に
  コピーできます (v0.6.2〜)

### 検索履歴・お気に入り・最近使ったフォルダ/ファイル
- キーワード欄はプルダウン式で、過去に検索したキーワードをすぐ呼び出せる
- 入力中の文字列と同じ検索履歴がある場合、Deleteキーでその履歴を削除可能 (v0.5〜)
- よく使うフォルダは☆ボタンでお気に入り登録し、プルダウンから呼び出せる
- 最近検索したフォルダ、最近開いたファイルもそれぞれ履歴から選択・再利用できる (v0.5〜)
- キーワード欄の候補一覧に、検索履歴に加えてインデックス内のファイル名・フォルダ名も
  表示されるようになりました (v0.6.6〜。設定でON/OFF可能)

### 簡易プレビュー・ハッシュ値計算
検索結果のファイルを選択すると、画面下に内容を簡易表示(設定でON/OFF切り替え可能)
- テキスト系ファイル(txt・py・md・csvなど): 中身の先頭部分を表示
- 画像ファイル(jpg・pngなど): サムネイル表示
- 他のアプリが使用中でアクセスできない場合や、ファイルが見つからない場合は、
  原因がわかりやすいメッセージを表示します
- HEIC/AVIF画像のプレビューに失敗した場合は、追加ライブラリ(pillow-heif)の
  インストールが必要な可能性がある旨をヒントとして表示します
- プレビュー欄の「ハッシュ値を計算」ボタンから、MD5/SHA-1/SHA-256のハッシュ値を
  計算・表示・コピーできます。バックグラウンドで計算するため、大きいファイルでも
  操作が固まることはありません (v0.6.2〜)

### 検索結果のエクスポート
検索結果一覧をCSVファイルとして保存できます(v0.4より、内容検索でヒットした
箇所もあわせて出力されます)。
- 更新日時をExcelで開いた際に「####」と表示されてしまう問題を修正しました (v0.6.6〜)
- 結果一覧表示用のフォルダ/ファイルアイコンがファイル名に混入し、Excelで文字化けして
  しまう問題を修正しました。アイコンを含まない元のファイル名で出力されます (v0.6.6〜)

### 検索速度表示・キーボードショートカット (v0.5〜)
- 検索完了時に検索速度(件/秒)を表示
- Escキーで検索停止、F5キーで再検索、Ctrl+Lで検索フォルダ欄へフォーカス、
  Ctrl+Fでキーワード欄へフォーカス
- 検索結果の件数表示を太字化して見やすく表示

### ウィンドウ幅に応じたレイアウト (v0.5〜)
お気に入り・最近のフォルダ・保存した検索条件や、各種チェックボックス・ボタンは、
ウィンドウの幅に応じて自動で折り返されます。ウィンドウを小さくしても項目が
画面からはみ出さず、スクロールなしでそのまま操作できます。

### ダークモード
設定からライト/ダークのテーマを切り替え可能です。メニュー・スクロールバー・
ドロップダウンリスト・右クリックメニューまで、画面全体にきちんと配色が反映されます。

### 設定の記憶
検索フォルダ・各種チェックボックス・ウィンドウサイズ・簡易プレビューやテーマの設定は、
次回起動時にもそのまま復元されます。保存した検索条件・お気に入り・各種履歴も
同様にお使いのPC内に保存されます。

### 高速・安全な内部設計
- 検索結果はまとめて画面に反映されるため、大量ヒット時も動作が軽快
- 検索結果が多い場合は自動的に「仮想リスト表示」に切り替わり、画面に表示しきれる
  範囲だけを描画することで、数万件規模の結果でもスクロールが軽快に動作 (v0.6.1〜)
- 検索結果を画面に反映する際のバッチサイズ・更新間隔は、ご利用の環境(CPU・メモリ)に
  応じて自動的に調整 (v0.6.1〜)
- アクセスできなかったフォルダの検出は、専用の事前走査ではなく検索・インデックス作成
  そのものの走査に統合。二重に走査することがなくなり、検索開始が速くなりました (v0.6.3〜)
- インデックスの更新は変更・新規ファイルのみを書き込み、消えたファイルのみを削除する
  差分方式。ディスクへの書き込み量を抑えつつ、途中で更新を停止した場合も未走査分を
  誤って削除しないよう安全に処理します (v0.6.5〜)
- インデックスデータベースの暗号化・復号は、途中で強制終了しても壊れないよう安全に処理
- 大量のファイルがあるフォルダでも、メモリ使用量を抑えて走査
- シンボリックリンクやWindowsのジャンクションが循環参照になっていても、無限ループにならないよう対策済み
- macOS/Linuxでファイルを開く際、コマンド自体は起動できても実際には失敗している場合
  (対応アプリが無い等)にきちんとエラーを表示するようコード上で対応
  (※本アプリはWindowsでのみ動作確認をしており、macOS/Linuxでの実機確認はできていません)
- データベース最適化(VACUUM)実行時、バックグラウンド自動インデックス更新と重ならないよう
  事前にチェック。一時的な競合で失敗した場合も、原因が伝わりやすいメッセージを表示
- 重複ファイル検索は、同一サイズのファイルのみハッシュ計算する2段階方式で無駄な処理を削減 (v0.6〜)
- 重複ファイル検索は、Windowsやアプリのシステムフォルダを検索対象にできない設計にして
  誤削除のリスクを低減 (v0.6.2〜)

### プライバシー第一設計
- ネットワーク通信は一切なし。オフライン環境でも実行可能
- インデックスデータベースはAES暗号化して保存。アプリを閉じると自動的に暗号化されます
- 設定・検索履歴・お気に入りフォルダ・保存した検索条件も、すべてお使いのPC内にのみ保存されます
- ファイル内容検索・重複ファイル検索も、お使いのPC内で完結して処理されます (v0.6〜)

## 注意事項

ダウンロード時に、ブラウザ(Chrome・Edgeなど)が「このファイルは危険な可能性があります」といった警告を表示することがあります。これは新しく・利用者の少ないファイルに対してブラウザが表示する一般的な確認です。「詳細」→「保持する」(または「このまま残す」)を選択してダウンロードを続行してください。

初回起動時に「WindowsによってPCが保護されました」といった青い警告が表示されることがあります。これは未署名の個人配布アプリで一般的に見られる確認画面です。「詳細情報」→「実行」で起動できます。

ウイルス対策ソフトによる誤検知が発生する場合があります。使用しているセキュリティソフトごとに回避方法が異なりますので、お使いのソフト名と合わせて検索し、回避方法をご確認ください。

「重複ファイル検索」でのファイル削除は元に戻せません。実行前に対象のファイルを必ずご確認ください。
